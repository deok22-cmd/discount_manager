# 텐트깔끄미 할인권 발행 시스템 (Service Proposal)

## 1. 개요
텐트깔끄미 서비스의 마케팅 효율을 높이기 위해, 특정 이벤트(예: 렌탈 고객 대상 할인)별로 고유한 할인권을 발행하고 사용 여부를 관리하는 시스템입니다.

---

## 2. 데이터베이스 필드 설계

### 2.1 이벤트(Events) 테이블
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| event_id | PK | 고유 식별자 |
| title | String | 이벤트 명 (예: 헬레녹스 고객 할인) |
| content | Text | 상세 내용 (고객 안내용) |
| discount_amount | Int | 할인 금액 |
| expiry_days | Int | 발급 후 유효 기간(일) |
| msg_template | Text | 발송용 기본 문구 |

### 2.2 할인권(Coupons) 테이블
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| coupon_id | PK | 고유 식별자 |
| event_id | FK | 소속 이벤트 ID |
| code | String | 쿠폰 번호 (예: TK-XJ82L1) |
| customer_name | String | 고객명 |
| phone | String | 전화번호 |
| status | Enum | 상태 (ISSUED / USED / EXPIRED) |
| issued_at | DateTime | 발행 일시 |
| expires_at | Date | 만료 일시 |
| used_at | DateTime | 사용 완료 일시 |

## 3. 주요 화면 및 기능

### 3.1 로그인 화면
- **단일 관리자 계정**
  - ID: `deok22`
  - PW: `misteam!01`

### 3.2 이벤트 관리 화면
- **이벤트 등록/수정**: 새로운 케어 프로모션 등록.
- **쿠폰 발행 현황**: 이벤트별 발급된 쿠폰 목록 확인.

### 3.3 할인권 발행 프로세스
1. **이벤트 선택**: 발행하고자 하는 이벤트 선택.
2. **대상 입력**: 고객명, 전화번호 입력.
3. **번호 생성**: `AA`로 시작하는 랜덤 8자리 코드 자동 생성. (예: `TK291048`)
4. **결과 메시지 생성**: 
   - "[텐트깔끄미] {이벤트명} 쿠폰이 도착했습니다. 코드: {번호}, 유효기간: {날짜}"
   - 관리자가 클릭 한 번으로 복사하여 문자/카톡 발송 가능.

### 3.4 할인권 조회 및 사용 처리
1. **코드 입력**: 고객이 제시한 쿠폰 번호 입력.
2. **검증**:
   - 존재 여부 확인 (없으면 "유효하지 않은 번호입니다" 경고)
   - 유효기간 확인 (만료 시 "사용 기간이 지났습니다" 경고)
   - 중복 사용 확인 (이미 사용된 경우 "이미 사용된 쿠폰입니다" 경고)
3. **정보 출력**: 조회 성공 시 이벤트명, 고객명, 전화번호, 만료일 노출.
4. **사용 완료**: [사용 처리] 버튼 클릭 시 상태 업데이트 및 할인 적용 기록 저장.

## 4. UI/UX 컨셉
- **색상**: 포레스트 그린 (#2E7D32) & 화이트 (깔끔하고 신뢰감 있는 무드)
- **사용 편의성**: 모든 데이터 조회는 실시간 비동기(AJAX) 처리로 화면 새로고침 없이 동작.
- **모바일 대응**: 관리자가 현장에서도 폰으로 쉽게 코드를 조회할 수 있도록 반응형 대시보드 구축.
